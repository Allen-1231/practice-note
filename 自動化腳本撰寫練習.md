# 自動化腳本撰寫練習

此指令可用於檢查語法而不執行。
```bash=
bash -n 檔名
```

### 測試是否違反最小權限原則: 檢查是否有不該擁有執行權限的檔案

:::info
`-x`：執行權限。
`-e`：exist，存在性，路徑中是否有任何東西，若否代表路徑不存在。
`-f`：regular file，檔案必須存在，是否為檔案。
`-d`：director，目錄必須存在，是否為目錄或目錄是否存在。
`-s`：檔案大小是否小於 0，檢查是否為空檔案。
`-w`：檔案是否可以寫入 (具有 `w` 權限)。

`if [ -f "$file" ]`：如果是檔案。
`if [ ! -f "$file" ]`：如果不是檔案。(`!` 與其他相鄰內容一樣都要空格)
:::

🎓學習到的內容：
- 有許多可以判斷是否為...的方法，像 `if [ ! -x "$file" ]` 可以判斷變數 file 的各檔案是否不可執行。
- 可以新增顏色。顏色為亂碼，最好的實作是先存放在變數中，其中 `NC` 為重設顏色、無顏色，無論如何都要加，以 `$NC` 作為要設定顏色的內容的斷點。

    在 `echo` 使用時必須加上 `-e` 參數，否則會直接把顏色代碼印出來。
    
    - `\033`：轉義，並且 `\033` 是最通用的八進位寫法。
    - `[`：指令的開始。
    - `0`：樣式。0 為正常；1 為加粗；4 為底線。
    - `;`：用來隔開樣式和顏色。
    - `31`...：顏色。31 為紅色；32 為綠色；33 為黃色；34 為藍色。
    - `m`：結束標誌。
- 使用 `-x` 而不直接執行看能否執行，能避免掉執行到惡意檔案的風險。
:::

#### Python 自動化腳本

另外，由於我在 Python 沒學過 sys 與 os 模組，加上想要藉由此機會練習看看將這個自動化腳本轉為 Python 的寫法，能在學習的同時，也訓練在 Python 寫自動化腳本處理系統的能力。

sys 與 os 是 Python 與底層系統之間的橋樑。os 讓 Python 具備管理員權限，可以執行新增資料夾、移除檔案、`cd`、`ls` 等動作；sys 讓 Python 可以接收使用者從終端傳進來的參數。

:::info
寫法差異：
| 功能               | Bash                         | Python            | 補充                                          |
|:------------------ |:---------------------------- |:----------------- |:--------------------------------------------- |
| 加法累加           | ((a++))、((a=a+1))、((a+=1)) | a=a+1、a+=1       | Bash 在處理數字運算時要使用雙括號 (( ))       |
| 指定運算           | num=$((1+1))                 | num = 1 + 1       | $(( )) 用來取得運算後的結果。                 |
| if 條件語句        | if [ ... ]; then ... fi      | if ...:           |                                               |
| if not             | if [ ! ... ]                 | if not ...:       |                                               |
| for 迴圈           | for x in $list; do ... done  | for x in list:    |                                               |
| 退出程式           | exit 1                       | sys.exit(1)       | 告訴系統程式跑完了，通常 0 代表成功。         |
| 參數 -- 腳本名稱   | $0                           | sys.argv[0]       | sys.argv 為參數列表，包含腳本名稱 (索引 0)。  |
| 參數 -- 第一個參數 | $1                           | sys.argv[1]       |                                               |
| 參數 -- 參數總數   | $#                           | len(sys.argv) - 1 | Python 會連腳本名一起算進去，然而 Bash 不會。 |
| 參數 -- 所有參數   | $@                           | sys.argv[1:]      |                                               |

---

| 功能         | Bash           | Python                    |
|:------------ |:-------------- |:------------------------- |
| 路徑是否存在 | [ -e "$path" ] | os.path.exists(path)      |
| 是否為檔案   | [ -f "$path" ] | os.path.isfile(path)      |
| 是否為目錄   | [ -d "$path" ] | os.path.isdir(path)       |
| 是否可執行   | [ -x "$path" ] | os.access(path, os.X_OK)  |
| 檔案是否為空 | [ -s "$path" ] | os.path.getsize(path) > 0 |
| 建立資料夾   | mkdir dirname  | os.mkdir("dirname")       |
| 執行系統指令 | ls             | os.system("ls")           |

---

| 功能                     | Bash                             | Python                         | 補充                                      |
|:------------------------ |:-------------------------------- |:------------------------------ |:----------------------------------------- |
| 字串拼接                 | full="\${var1}_\${var2}"         | full = f"{var1}_{var2}"        | f-string 是 Python 目前最主流的寫法。     |
| 讀取使用者輸入           | read var_name                    | var_name = input()             | Bash 用 read；Python 用 input()           |
| 指令結果轉變數           | result=$(ls)                     | result = os.popen('ls').read() | Python 比較少這樣做，通常會調用專門模組。 |
| 長度計算                 | ${#var}                          | len(var)                       | Bash 中在變數名稱前面加上 #，代表數量。   |
| 條件判斷 -- AND          | if [ A -a B ]、if [[ A && B ]]   | if A and B:                    |                                           |
| 條件判斷 -- OR           | if [ A -o B ]、if [[ A \|\| B ]] | if A or B:                     |                                           |
| 條件判斷 -- 檢查字串相等 | if [ "\$a" = "$b" ]              | if a == b:                     | Bash 用單個 = 或雙個 = 皆可。             |
| 條件判斷 -- 檢查變數為空 | if [ -z "$var" ]                 | if not var:                    |                                           |

---

| 功能         | Bash                         | Python                            | 補充                                                                                                             |
|:------------ |:---------------------------- |:--------------------------------- |:---------------------------------------------------------------------------------------------------------------- |
| 固定次數迴圈 | for i in {1..5}; do ... done | for i in range(1, 6)              | Bash 的 {1..5} 表示 1 2 3 4 5; {A..Z} 表示 A 到 Z; {1..10..2} 表示 1 3 5 7 9，這等同於 Python 的 range(1, 11, 2) |
| 無窮迴圈     | while true; do ... done      | while True: ...                   |                                                                                                                  |
| 定義函數     | my_func() { ... }            | def my_func():                    |                                                                                                                  |
| 函數傳參     | my_func "arg1"               | my_func("arg1")                   | Bash 的參數在函數內是用 $1 拿。                                                                                  |
| 回傳值       | return 0 (狀態碼)            | return value                      | Bash 的 return 只能傳回 0-255 的數字。                                                                           |
| 印出且不換行 | echo -n "Hello"              | print("Hello", end="")            | Python 的 print 預設自帶換行。                                                                                   |
| 彩色輸出     | echo -e "\033[32mOK\033[0m"  | (需 import colorama 或使用轉義符) | Bash 處理顏色的方式比較底層且直接。                                                                              |

break 與 continue 兩者皆完全一樣。


:::

Bash 以字串型態為主，除非用 (())，否則所有東西型態都是字串。

Python 是物件導向，數字是 int，文字是 str，有明確的資料型態分別。若要數字與字串相加，必須要手動轉型別，如 str(number)。

第 20 行與 28 行的 for 迴圈中，sys.argv[1] 為一個字串，因此會將這個路徑假如是 "/etc"，在迴圈就會被拆成 `/`、`e`、`t`、`c`。

用 `os.listdir()` 拿到的檔名，會因為它只是一個名稱、字串，而失敗。
修正：需將資料夾路徑與檔名結合 --> os.path.join(sys.avgr[1], file)。

修正第 31 行：來源路徑的 file 只是檔名，因此也要指定從 `sys.argv[1]` 這個資料夾去找。目標路徑直接寫 "backup_sh" 即可。
